Getting Started

This section provides an overview of how to begin using the CDA Profiler tool.

Importing Data

To bring data into the CDA Profiler, navigate to Import Data in the top menu. Select the data source you wish to import from (e.g., file upload, database, or another external source).

  • Number of Documents to Import(database import only): Specify how many documents you want to process in one go.
  • Filter ID(database import only): Optionally provide a filter or identifier to narrow down which records to import.
  • Profile Name: Give a unique name to the import session, especially if you plan to run multiple imports.

After configuring these fields, click Start Import. The tool will parse the incoming data using XSLT transformations and XPath expressions. If you encounter issues, check the Troubleshooting section below.

Generating Reports

Once data is imported, you can view your custom reports in the Reports section.

  • Reports: Choose from any of the reports you have created and view the data by section and field.
  • Exporting: Export generated reports to an Excel worksheet for sharing or further analysis.
  • Data Quality Summaries: Not available at this time.
  • Validation Results: Not available at this time.

The reports can help you spot trends, ensure compliance with standards, and validate document accuracy.

System Settings

The Settings page lets administrators configure the CCDA Profiler:

  • XSLT / XPath Expressions: Edit or upload transformations for extracting specific data from CCD documents. Use the testing utility to test your changes against a document(file must be on your local).
  • Deleting Data: You may delete a profile and its associated data or delete all data. Note, this action cannot be undone.

Any changes here can significantly impact how the profiler parses data and displays reports, so proceed with caution.

Troubleshooting / FAQ

Below are common issues and tips:

  • Data Import Errors: Ensure your file format is valid. Check console or server logs for details on parsing failures.
  • Missing Reports: Confirm the import completed successfully. If the data was invalid, the report might not generate.
  • XSLT Issues: Validate your XSLT / XPath syntax. Incorrect expressions can cause partial or empty transformations.
  • Permission Denied: If you can't access a page or feature, verify your user role under User Roles & Permissions.
  • General Performance: Large imports can take time. Try splitting files or allocating more resources to the server.

If you still experience problems, contact your system administrator or consult the application logs for detailed error messages.
